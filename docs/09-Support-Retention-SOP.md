# Support and Retention SOP

## Objective

Keep clients retained by making the service feel reliable, responsive, and easier than alternatives. The AI agent handles Tier 1 support and proactive retention; the human handles escalations.

## Retention principle

Clients stay when the service is simple, visibly helpful, and low-maintenance. They leave when support feels vague, slow, or unpredictable. The agent's job is to make the service feel like it's always on.

## Support channels

Primary: Email (agent-monitored)
Secondary: Lightweight ticket form or WhatsApp Business

## Agent-driven support

### Tier 1: Agent handles automatically
- Minor content edits (text swaps, image changes)
- Contact detail updates
- New testimonial additions
- Chatbot answer updates
- Billing questions (plan info, next renewal date)
- FAQ responses
- Monthly report delivery

### Tier 2: Agent drafts, human approves
- Design or layout changes
- New page creation
- Anything affecting site structure
- Requests that might be out of scope

### Tier 3: Human handles directly
- Angry or frustrated clients
- Legal or compliance questions
- Custom feature requests outside any package
- Disputes or cancellation negotiations
- Any message containing threats or legal language

## Agent support guardrails

The agent MUST NOT:
- Offer refunds, discounts, or free months without human approval
- Promise custom features or scope changes
- Respond to messages containing legal threats
- Delete or modify content it is not certain about
- Escalate a client's frustration by being repetitive or dismissive

The agent MUST:
- Acknowledge every request within 4 hours
- Classify the request type and priority
- Resolve Tier 1 items within 24 hours
- Escalate Tier 2 and Tier 3 to the human with full context
- Log every interaction in the client's support history

## Response policy

| Priority | Response Time | Resolution Time | Handled By |
|---|---|---|---|
| Critical (site down, broken forms) | < 4 hours | < 24 hours | Agent detects → Human fixes |
| Standard (content edit, update) | < 24 hours | < 48 hours | Agent |
| Low (monthly report, suggestion) | < 48 hours | Next report cycle | Agent |

## Monthly retention actions (Agent-operated)

Every month, the agent sends each client at least one visible proof of value:

- Analytics summary (auto-generated)
- One improvement recommendation
- Small proactive enhancement (if within scope)
- Chatbot performance note
- Site issue detected and fixed before the client noticed

## Churn signal detection

The agent monitors for churn signals and alerts the human:

- No support requests or engagement for 60+ days
- Repeated price sensitivity in messages
- Repeated requests outside scope
- Billing failures
- Long client silence after launch
- Complaints about unclear value
- Client viewing cancellation page or asking about leaving

## Churn prevention moves

When the agent detects a churn signal:

1. Agent sends a personalized retention message (value reminder, recent improvements)
2. If the client responds negatively, agent escalates to human
3. Human may offer a lower tier, a pause, or a direct conversation
4. If the client cancels, agent handles offboarding

## Offboarding (Agent-operated)

If a client cancels:

1. Agent confirms the effective cancellation date.
2. **Follow the cancellation and domain transition procedure in doc 08** (7-day grace period, content export offer, domain transfer rules, site removal timeline).
3. Agent clarifies whether content export is provided (yes — offered upon request per doc 08).
4. Agent revokes dashboards and support access at the correct time.
5. Agent marks the account status clearly (Cancelled + reason).
6. Agent logs the cancellation reason.

## Support boundaries

The agent enforces these boundaries consistently:

- Support is not strategy consulting
- Support is not copywriting from scratch
- Support is not marketing management
- Support is not custom development
- Any of the above = separate pricing or polite decline
