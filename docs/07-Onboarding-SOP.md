# Client Onboarding SOP

## Objective

Turn a newly paying client into a fully configured managed website account with minimal friction and clear expectations. The AI agent handles the entire onboarding flow; the human reviews the final output before launch.

## Onboarding goals

- Confirm scope
- Collect materials quickly
- Avoid delivery ambiguity
- Launch fast
- Set support boundaries early

## Onboarding checklist

Agent collects the following from the client:

- Business legal / trading name
- Contact person
- Best email and phone
- Logo files if available
- Brand colors if any
- Service list
- Service areas / locations
- Existing photos
- Existing copy or brochure text
- Testimonials / reviews to use
- Preferred CTA (call, form, WhatsApp, booking link)
- Social links
- Required legal pages if any

## Agent-driven onboarding workflow

1. Payment confirmed (via Dodo/Polar checkout).
2. Agent sends welcome message with intake form or guided questions.
3. Agent collects materials from the client.
4. Agent sends one follow-up if materials are missing after 48 hours.
5. Agent assembles site from niche template using collected + publicly available content.
6. Agent configures analytics, chatbot, and CMS fields.
7. Agent runs automated QA checks (mobile layout, forms, links, speed).
8. **Human reviews the site before launch.**
9. Agent launches the site or sends preview link per approval model.
10. Agent sends credentials, support instructions, and billing reminder to the client.

## Human touchpoints in onboarding

| Step | Operator | Purpose |
|---|---|---|
| Payment confirmation | Agent (automated) | — |
| Welcome message | Agent | — |
| Material collection | Agent | — |
| Site assembly | Agent | — |
| QA checks | Agent (automated) | — |
| **Final review before launch** | **Human** | **Quality gate** |
| Launch | Agent | — |
| Handoff message | Agent | — |

## Timeboxing rule

Do not let onboarding drag indefinitely.

- If client does not send materials within 72 hours, agent builds with publicly available content and notes assumptions.
- If the client becomes unresponsive, agent pauses nonessential customization and notifies the human operator.
- Keep subscription terms clear regarding inactivity.

## Welcome message contents

The agent's welcome message should state:

- Thank you and confirmation of plan
- What happens next
- What materials are needed (with a simple list)
- Expected launch window (3–7 business days)
- Where to send support requests

## Minimum viable launch content

A site can usually launch with:

- Business name and contact details
- Service list
- Service area
- 3–5 trust points or testimonials
- Strong CTA
- A few clean images

If the client has not provided these, the agent sources them from the client's existing public presence (current website, Google Business listing, social profiles).

## Approval policy

Recommended model for agent-operated delivery:

- Agent assembles and runs QA automatically
- Human does a final quality review before publishing
- Launch-after-window model: if the human has not flagged issues within 24 hours of preview, the agent proceeds to launch
- **The 24-hour window IS the human quality gate** — the operator is expected to check previews within 24 hours. If absent for longer periods (vacation, sick, travel), the operator should pause the onboarding pipeline before going offline.

## Post-launch handoff

Agent sends:

- Live site URL
- How to request edits (simple instructions)
- Billing reminder (next renewal date)
- What is included each month
- Support window and response expectations
