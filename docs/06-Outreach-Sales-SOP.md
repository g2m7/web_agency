# Outreach and Sales SOP

## Objective

Convert qualified bad-website prospects into paying monthly retainer clients using a 2-step, agent-operated, async sales process.

## Sales philosophy

The sale should feel light, specific, and low-friction. The prospect should not need a call to understand the value. Demos are reserved for interested prospects only — not blasted to every lead. This saves compute and filters out tire-kickers before the agent invests in building a mockup.

## The 2-step sales motion

### Step 1: Hook email (agent sends to all qualified leads)

The first email does NOT include a demo. It highlights one visible issue, shows relevance, and asks one question: "Want to see a better version?"

- Keeps the email short and high-response
- No compute wasted building demos for leads that never reply
- Natural qualification — only interested prospects proceed

### Step 2: Demo + pricing email (agent sends only on positive reply)

When a prospect replies interested, the agent builds a tailored demo using the niche template + the prospect's real business content, then sends it with pricing and a checkout link.

- Higher demo quality (fewer demos = more care per demo)
- Higher close rate (demo is a reward for engagement, not a cold attachment)
- Natural momentum — the prospect is already engaged

## Core sales flow

1. Agent selects a hot lead from the scored list.
2. Agent drafts a short personalized hook email highlighting one visible issue.
3. Agent sends the email.
4. Agent follows up on a fixed sequence.
5. If prospect replies interested, agent builds a demo and sends it with pricing.
6. Agent handles objections and answers questions.
7. Agent sends checkout link and confirms payment.

## The first email must do three things

- Show relevance (mention their business, niche, city)
- Mention one visible issue (specific, undeniable)
- Ask one question: "Want me to show you a better version?"

## The second email (demo email) must do four things

- Deliver the demo link
- Explain what is included in the monthly plan
- State the price clearly
- Provide a checkout link as the next step

## Outreach rules

- Keep the hook email short — under 80 words.
- Mention only one or two issues.
- Do not insult the current site.
- Do not use agency buzzwords.
- Do not ask for a call in the first email.
- Do not include a demo or link in the first email.
- Do not send large attachments.
- Always end the hook email with a simple question.

## Recommended sequence

### Email 1 — Hook
Purpose: show relevance, highlight one issue, ask if they want to see a better version.
No demo. No link. Just one question.

### Follow-up 1 (3–4 days later)
Purpose: brief bump, restate the issue, ask again.
Still no demo — just re-offer to show one.

### Follow-up 2 (7 days later)
Purpose: final check-in, mention the issue one more time, easy yes/no reply.

### If prospect replies "yes" or "interested"
Agent builds the demo using the niche template + their business content.
Agent sends demo email with:
- Demo link
- What is included in the plan
- Price
- Checkout link

### If prospect replies with questions or objections
Agent classifies the objection and responds using the objection handling guide below.

### If no reply
Complete the sequence and archive the lead.

## Objection handling

### If price objection
- Reframe around management + ongoing support
- Offer the lower package if available
- Do not offer discounts unless human-approved

### If they ask "Is this custom?"
- Explain that the site is adapted to their business, but built on a proven structure for faster launch and lower cost.

### If they ask for a call
- Offer a quick 5-minute phone or SMS walkthrough.
- Do not block the sale on a call — keep it optional.

### If they ask "Why should I trust you?"
- Point to the demo built with their own content.
- Mention cancel-anytime policy.
- Keep it brief and confident.

## Agent guardrails for sales

The agent MUST NOT:
- Offer discounts, free months, or price reductions without human approval
- Promise scope changes or custom features outside the package
- Send a demo before the prospect expresses interest
- Respond to angry or legal-sounding messages (escalate to human)
- Make guarantees about revenue, SEO rankings, or lead volume
- Share other clients' data or demos

The agent MUST:
- Use the prospect's real business name and city in every email
- Reference a specific visible issue from the audit
- Include checkout link only after interest is confirmed
- Log every interaction, reply, and objection
- Classify every reply (interested / not interested / objection / question)

## No-call operating stance

Default stance:
- Close async via email whenever possible.
- Offer a quick 5-minute call or SMS walkthrough as an option, not a requirement.
- Track whether certain niches consistently require calls — this is a signal to adjust the approach or the niche.

## Close criteria

A lead is considered closed when:
- They agree to the package in writing
- They complete payment
- Onboarding is initiated

## Deal desk rules

Before taking payment, the agent confirms:
- Chosen package
- Monthly price
- What is included
- Expected launch timeline
- Who provides missing content if needed
- Cancellation terms

## Sales tracking fields

Track:
- Lead source
- Niche
- Date first contacted
- Follow-up count
- Reply type (none / interested / not interested / objection)
- Demo sent (yes/no and date)
- Objections raised
- Package discussed
- Closed / lost reason
- Agent confidence score on the lead

## Reasons deals are lost

Standardize reasons:
- No response
- Too expensive
- Already has provider
- Not a priority
- Wants call-heavy process
- Needs custom build outside scope
- Low trust / unclear fit
- Requested demo but did not convert

## Why the 2-step motion works

| Factor | Demo-for-everyone | Demo-on-request |
|---|---|---|
| Emails sent per batch | 100 | 100 |
| Demos built | 100 | 3–5 |
| Agent compute per lead | ~500 tokens | ~50 tokens |
| Demo quality | Generic | High (focused on interested prospect) |
| Close rate on demos | Low (cold) | High (warm lead) |
| Tire-kicker filtering | None | Built-in |
